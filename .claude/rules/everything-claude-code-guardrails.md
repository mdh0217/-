# Everything Claude Code 가이드라인

ECC Tools가 저장소 히스토리를 분석하여 생성한 파일입니다. 강제 정책으로 취급하기 전에 반드시 검토하세요.

## 커밋 워크플로우

- fix, test, feat, docs 등의 접두사를 사용하는 `conventional` 커밋 메시지를 권장합니다.
- 새로운 변경사항은 저장소의 기존 풀 리퀘스트 및 리뷰 흐름에 맞게 유지하세요.

## 아키텍처

- 현재의 `hybrid` 모듈 구성 방식을 유지하세요.
- 현재 테스트 레이아웃(`separate`)을 따르세요.

## 코드 스타일

- 파일 이름에 `camelCase`를 사용하세요.
- `relative` 임포트와 `mixed` 익스포트를 권장합니다.

## ECC 기본값

- 현재 권장 설치 프로파일: `full`.
- PR에서 위험한 설정 변경을 검증하고, 설치 매니페스트를 소스 컨트롤에 유지하세요.

## 감지된 워크플로우

- database-migration: 마이그레이션 파일을 포함한 데이터베이스 스키마 변경
- feature-development: 표준 기능 구현 워크플로우
- add-language-rules: 코딩 스타일, 훅, 패턴, 보안, 테스트 지침을 포함한 새 프로그래밍 언어를 규칙 시스템에 추가

## 검토 알림

- 저장소 컨벤션이 크게 바뀌면 이 번들을 재생성하세요.
- 예외 처리(suppressions)는 좁고 감사 가능하게 유지하세요.
